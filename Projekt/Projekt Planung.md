# Checkliste – System Migration für IST Aufnahme

## Hardware / Infrastruktur
- [ ] CPU (Anzahl Kerne, Modell)
- [ ] RAM (Größe, Auslastung)
- [ ] Disk (Größe, Typ, Auslastung)
- [ ] Netzwerkinterfaces und IP-Adressen
- [ ] Virtualisierungsplattform und Version

## Betriebssystem
- [ ] OS-Version und Kernel
- [ ] Patchstand
- [ ] Installierte Pakete / Dienste
- [ ] Laufende Prozesse und Abhängigkeiten
- [ ] Cron-Jobs und geplante Tasks
- [ ] Benutzer, Gruppen, sudo-Berechtigungen

## Applikation
- [ ] Softwareversion(en)
- [ ] Konfigurationsdateien und Pfade
- [ ] Abhängigkeiten (Libraries, Frameworks)
- [ ] Customizations / Code-Anpassungen
- [ ] Datenbankschema und -größe

## Netzwerk & Sicherheit
- [ ] Firewall-Regeln (iptables / firewalld)
- [ ] Offene Ports / Dienste
- [ ] SSL-Zertifikate (Aussteller, Ablaufdatum)
- [ ] DNS-Einträge
- [ ] Zugriffskontrollen (SSH-Keys, API-Keys)

## Daten
- [ ] Datenmenge und Wachstumsrate
- [ ] Backup-Strategie (Wo, Wie oft, Retention)
- [ ] Letztes erfolgreiches Backup
- [ ] Speicherorte von Uploads / Attachments

## Schnittstellen & Integrationen
- [ ] Externe Systeme (APIs, LDAP, Mail-Server)
- [ ] Interne Abhängigkeiten (andere Server, Datenbanken)

## Betrieb
- [ ] Wartungsfenster / erlaubte Downtime
- [ ] SLAs und Verfügbarkeitsanforderungen
- [ ] Verantwortliche Personen / Ansprechpartner
